# Donggri Ledger Build Story Step 6

## Step 6. 해외 주식과 ETF까지 감당하는 멀티통화 투자 구조로 개편하기

### 메인 키워드

- 해외 주식 가계부 만들기
- 해외 ETF 수익률 계산
- 멀티통화 투자 포트폴리오

### 검색 의도

- 미국 ETF나 해외 주식을 가계부에 넣었더니 현재가가 10원처럼 보이거나 수익률이 -99%로 깨지는 이유를 알고 싶은 사람
- 원화 기반 가계부에서 달러 자산과 원화 자산을 함께 관리하는 구조를 찾는 사람

### 메타 제목

- 해외 주식과 ETF까지 감당하려면 가계부 투자 모델을 멀티통화로 바꿔야 하는 이유

### 메타 설명

- BITO 같은 미국 ETF를 예로 들어, 원화 단일 구조에서 왜 수익률이 깨지는지와 `market`, `currency`, `fx_rate_krw` 기준으로 투자 모델을 다시 설계한 이유를 정리합니다.

### 도입부

투자 기능을 붙이고 나면 국내 자산만 관리할 때는 멀쩡하던 숫자가 해외 종목에서 바로 무너집니다. 실제로 미국 ETF인 BITO를 넣었을 때 현재가가 10원처럼 보이고 수익률이 -99.96%로 찍히는 문제가 나왔습니다. 가격을 못 불러온 것이 아니라, 달러 가격을 원화처럼 다뤄버린 데이터 구조가 문제였습니다. 이 시점부터는 단순한 시세 연동보다 투자 모델 자체를 다시 설계해야 했습니다.

### 본문

처음 투자 기능은 `평균매수가`, `현재가`, `수익률`만 있으면 충분해 보였습니다. 하지만 해외 주식과 ETF, 코인을 함께 넣는 순간 이 구조는 한계를 드러냅니다. 미국 ETF의 현재가는 달러인데 앱은 모든 숫자를 원화처럼 렌더링했고, 사용자는 평균매수가를 원화 또는 달러 중 아무 기준으로나 넣게 되면서 계산 기준이 무너졌습니다. 그래서 Step 6에서는 투자 데이터를 `종목 유형` 중심이 아니라 `시장 + 통화` 중심으로 다시 정리했습니다.  

핵심은 네 가지입니다. 첫째, `market`과 `currency`를 투자 데이터에 추가해 국내 자산과 해외 자산을 같은 테이블에서 구분할 수 있게 했습니다. 둘째, `current_price`는 더 이상 원화 환산값이 아니라 원통화 가격을 저장하도록 바꿨습니다. 셋째, 원화 환산이 필요할 때는 `fx_rate_krw`를 별도로 저장해 요약 카드와 평가금 계산에만 사용하게 했습니다. 넷째, 실시간 연동 종목은 직접 심볼을 입력하는 대신 검색 결과에서 선택하도록 바꿨습니다. 이렇게 해야 `BITO`, `AAPL`, `005930.KS`처럼 시장별 표기 차이와 오타 문제를 동시에 줄일 수 있습니다.  

입력 UX도 함께 바꿨습니다. 실시간 종목은 `심볼 검색 -> 결과 선택 -> 추가` 흐름으로 넣고, 펀드만 수동 입력을 남겨뒀습니다. 표에서는 평균매수가와 현재가는 달러면 달러, 원화면 원화로 그대로 보여주고, 원금과 평가금은 원화 환산값을 함께 표시했습니다. 이 구조가 중요한 이유는 사용자가 숫자를 “왜 이렇게 보이는지” 직관적으로 이해할 수 있기 때문입니다. 수익률은 같은 통화끼리 비교하고, 전체 자산 합계는 원화로 모으는 방식이 결국 가장 덜 헷갈립니다.

### 코드 예시

```sql
alter table public.investments
    add column if not exists market text not null default 'MANUAL';

alter table public.investments
    add column if not exists currency text not null default 'KRW';

alter table public.investments
    add column if not exists fx_rate_krw numeric(20, 8) not null default 1;
```

### 체크 포인트

- 해외 ETF의 현재가가 더 이상 `10원`처럼 보이지 않는가
- 평균매수가와 현재가가 같은 통화 기준으로 비교되는가
- 투자 요약 카드에서는 원화 환산값이 안정적으로 보이는가
- 심볼 직접 타이핑보다 검색 후 선택 흐름이 더 덜 막히는가

### FAQ

- 왜 `해외 ETF` 타입만 따로 만들지 않았나요  
  핵심 문제는 유형보다 통화와 시장이기 때문입니다. 해외 ETF, 미국 주식, 코인은 결국 통화와 환율을 같이 다뤄야 안정적으로 계산됩니다.

- TradingView를 바로 붙이면 해결되지 않나요  
  차트에는 좋지만, 투자 금액 저장과 수익률 계산의 기준을 정리해주지는 못합니다. 먼저 데이터 모델을 바로 세우는 것이 우선입니다.

### 마무리

Step 6의 핵심은 시세 API를 바꾼 것이 아니라 투자 숫자가 망가지지 않도록 기준을 다시 세운 데 있습니다. 해외 자산을 다루는 순간 가계부는 더 이상 단일 통화 앱이 아니고, 입력 UX까지 포함해 멀티통화 구조를 의식해야 합니다. 이 단계를 지나면 투자 기능은 “보이는 기능”이 아니라 실제로 믿고 쓸 수 있는 기능에 가까워집니다.
