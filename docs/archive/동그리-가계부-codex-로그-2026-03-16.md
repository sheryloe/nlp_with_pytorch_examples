## 이번 구간의 핵심 결정

- 해외 주식/해외 ETF/코인을 `원화 강제 저장`이 아니라 `시장 + 통화` 구조로 다룬다.
- 투자 입력은 자유 텍스트보다 `심볼 검색 -> 결과 선택 -> 추가` 흐름으로 바꿔 입력 실수를 줄인다.
- 원통화 가격과 원화 환산 평가금은 분리해서 보여준다.
- Step 문서는 단순 구현 로그가 아니라 SEO형 블로그 원고 구조로 정리한다.
- GitHub Wiki, README, Notion, 세션 로그를 서로 끊기지 않게 동기화한다.

## 이번 구간에서 반영한 구현

### 투자 모델 개편

- `supabase/schema.sql`의 `investments`에 `market`, `currency`, `fx_rate_krw`, `price_source`를 추가했다.
- 해외 ETF/주식은 `USD`, 국내 자산은 `KRW`처럼 원통화를 분리해 저장하는 방향으로 정리했다.
- `refresh-market-prices`는 원통화 가격과 환율을 분리 저장하도록 수정했다.

### 심볼 검색 기반 입력

- `supabase/functions/search-market-symbols/index.ts`를 추가했다.
- 사용자는 임의 문자열을 넣는 대신 검색 결과에서 종목을 선택하도록 바꿨다.
- `BITO` 같은 해외 ETF도 시장/통화 정보와 함께 확인한 뒤 추가할 수 있게 했다.

### 화면과 문서

- 투자 입력/표 렌더링을 멀티통화 표시 기준으로 정리했다.
- Step 6 원고를 `docs/BUILD_STORY_STEP6.md`로 작성하고 Notion에 하위 페이지로 반영했다.
- GitHub Wiki에 `Build-Story` 문서를 추가하고 README에도 최신 Step 흐름을 연결했다.
