## 프로젝트 한 줄 요약

Donggri Ledger는 `Supabase + Vercel + GitHub` 구조로 운영하는 개인 가계부 서비스이며, 카드 결제예정, 고정지출, 예산, 투자 포트폴리오, 백업/복원을 한 흐름으로 관리하는 프로젝트입니다.

## 이번 대화에서 확정된 방향

- 로컬 FastAPI + SQLite 상시 운영이 아니라 `Supabase + Vercel` 서비스형 구조로 전환
- 사용자별 데이터 분리를 위해 `user_id`와 `RLS`를 적용
- 이메일 복구 대신 보안질문 기반 복구 흐름 채택
- GitHub Pages는 Notion/빌드스토리 허브가 아니라 제품 소개와 SEO 랜딩용으로 정리
- GitHub Wiki는 제품/기능/구조/운영/로드맵 문서 허브로 구성

## 핵심 구현 결과

### 서비스 구조

- 프론트: 정적 웹 앱
- 인증/DB: Supabase Auth + Postgres
- 보안: `RLS`
- 자동화: Supabase Edge Functions
- 배포: Vercel
- 문서/브랜딩: README, GitHub Pages, GitHub Wiki, Notion

### 인증과 계정

- 아이디 + 비밀번호 로그인
- 회원가입 시 보안질문 3개 저장
- 보안질문 기반 비밀번호 재설정
- 회원가입 50명 제한

### 가계부 기능

- 수입 / 지출 / 투자 거래
- 은행 / 현금 / 카드 / 투자 자산 구분
- 카드 결제예정과 결제통장 관리
- 고정지출
- 카테고리별 예산
- 투자 포트폴리오와 실시간 시세 갱신
- JSON 백업 / 복원
