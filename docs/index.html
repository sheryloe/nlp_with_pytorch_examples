Core feature set

가계부 기능을 서비스 단위로 다시 묶었습니다

입력 기능보다 실제 사용 흐름이 먼저 보이도록 다시 묶었습니다.

계정 접근

아이디 로그인, 보안질문 3개 기반 비밀번호 재설정, 50명 가입 제한.

거래와 자산

수입, 지출, 투자 거래를 입력하고 은행, 현금, 카드, 투자계좌를 분리 관리.

카드 결제예정

카드 자산별 결제일과 결제통장을 연결해 이번달 결제예정 금액을 계산.

고정지출과 예산

고정지출 등록과 카테고리별 예산 설정으로 생활비 흐름을 월 단위로 확인.

투자 포트폴리오

주식, 코인, ETF, 펀드를 분리 관리하고 평가금과 수익률을 추적.

데이터 안전장치

JSON 내보내기와 복원, 삭제 전 연결 데이터 검사, 운영 로그 관리.

썸네일 대신 메인 흐름이 보이는 전체 화면 중심으로 배치했습니다.

Architecture

서비스 구조와 코드 구조를 분리해서 설명합니다

운영 레이어와 저장소 레이어를 섞지 않고 역할만 짧게 나눴습니다.

Service stack
  • Frontend

    정적 웹 앱을 Vercel에 배포하고, 로그인 이후 UI와 상태 관리를 브라우저에서 처리합니다.

  • Auth / Database

    Supabase Auth와 Postgres, RLS로 사용자별 데이터 분리와 접근 제어를 담당합니다.

  • Automation

    Edge Functions로 회원가입, 보안질문 복구, 투자 가격 새로고침 같은 서버 보조 로직을 처리합니다.

Repository layout
`web/`

실제 UI, 화면 상태, 사용자 상호작용 로직.

`supabase/`

스키마, RLS, Edge Functions 정의.

`docs/`

GitHub Pages 랜딩, SEO 리소스, 스크린샷, 세션 로그.

`scripts/`

정적 웹 빌드와 배포용 보조 스크립트.

Recommended roadmap

다음 단계는 기능 추가보다 흐름 강화에 가깝습니다

FAQ

자주 묻는 질문

Donggri Ledger는 어떤 구조로 운영되나요?

프론트엔드는 Vercel, 인증과 데이터베이스는 Supabase, 코드와 문서는 GitHub 기반으로 운영합니다.

현재 바로 사용할 수 있는 기능은 어디까지인가요?

회원가입, 로그인, 카드 결제예정, 자산 관리, 고정지출, 예산, 투자 포트폴리오, 백업과 복원을 사용할 수 있습니다.

왜 카드 결제예정 기능이 중요한가요?

카드를 쓴 날과 실제 돈이 빠져나가는 날이 다르기 때문에, 생활비 흐름을 맞추려면 결제예정 금액을 따로 관리해야 합니다.

누가 쓰기 좋은 서비스인가요?

생활비, 카드 결제일, 투자 자산을 한 화면에서 함께 보고 싶은 개인 사용자에게 맞는 구조입니다.
