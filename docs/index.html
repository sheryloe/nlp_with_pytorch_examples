Current features

현재 제공하는 핵심 기능

이 페이지는 제품 소개와 코드 구조 설명에 집중합니다. 현재 서비스에서 바로 확인할 수 있는 기능만 정리했습니다.

계정 접근

아이디와 비밀번호 로그인, 보안질문 3개 기반 비밀번호 재설정, 50명 가입 제한을 지원합니다.

거래 관리

수입, 지출, 투자 거래를 입력하고 월별 흐름과 최근 내역을 한 번에 볼 수 있습니다.

자산 분리

은행, 현금, 카드, 투자계좌를 분리해 실제 재무 상태를 더 정확하게 관리합니다.

카드 결제예정

카드 자산별 결제일과 결제통장을 연결해 이번달 카드 결제예정 금액을 계산합니다.

고정지출과 예산

고정지출 등록과 카테고리별 예산 설정으로 생활비 초과 여부를 바로 확인할 수 있습니다.

투자 포트폴리오

주식, 코인, ETF, 펀드를 분리 관리하고 평가금과 수익률을 한 흐름으로 추적합니다.

백업과 복원

JSON 내보내기와 복원 기능으로 데이터 이전과 안전한 보관을 지원합니다.

검색과 점검

거래 검색, 리포트, 삭제 전 연결 데이터 검사 등 운영 마감 UX도 같이 정리되어 있습니다.

Product screens

서비스 화면을 크게 보는 섹션

작은 카드 캡처 대신, 현재 서비스의 핵심 화면을 전체 흐름이 보이도록 크게 배치했습니다.

Architecture

서비스 구조와 코드 구성

운영은 서비스 기준으로, 코드는 책임 기준으로 나눴습니다. GitHub Pages는 소개용 랜딩, 실제 서비스는 Vercel 배포를 전제로 잡고 있습니다.

Frontend

정적 웹 앱을 Vercel에 배포합니다. 로그인, 거래 입력, 자산 관리, 투자 화면이 이 레이어에 있습니다.

Auth / DB

Supabase Auth와 Postgres, RLS를 이용해 사용자별 데이터 분리와 접근 제어를 처리합니다.

Automation

Supabase Edge Functions로 회원가입, 보안질문 복구, 투자 가격 새로고침 같은 서버 보조 로직을 담당합니다.

Repository

GitHub에서 코드 이력, 문서, 세션 로그를 관리하고 GitHub Pages 정적 랜딩도 같이 유지합니다.

`web/`

실제 UI, 상태 관리, 브라우저 동작 로직이 들어 있습니다.

`supabase/`

스키마, RLS, Edge Functions와 같은 서비스 백엔드 정의가 들어 있습니다.

`docs/`

GitHub Pages 랜딩, SEO 리소스, 스크린샷, 세션 로그가 들어 있습니다.

`scripts/`

정적 웹 빌드와 배포용 보조 스크립트를 관리합니다.

Recommended roadmap

다음으로 붙이면 좋은 기능

지금 구조를 무너지지 않게 유지하면서, 실제 사용성과 운영 효율을 같이 올릴 수 있는 기능을 우선순위로 정리했습니다.

Priority 1

반복 수입과 반복 지출 자동 생성, 계좌 간 이체 거래를 붙이면 월간 입력 부담이 크게 줄어듭니다.

Priority 2

예산 초과, 카드 결제일, 고정지출 알림 센터를 만들면 실사용 감각이 더 올라갑니다.

Priority 3

CSV / Excel 가져오기와 PDF 리포트 내보내기로 데이터 이동성과 공유성을 강화할 수 있습니다.

Priority 4

투자 수익률 추이 차트와 가격 갱신 이력을 붙이면 포트폴리오 탭의 분석력이 더 좋아집니다.

Priority 5

가입 현황, 함수 상태, 오류 로그를 보는 운영 대시보드를 만들면 서비스형 운영이 더 편해집니다.

Future option

월마감 스냅샷, 목표 저축, 가족 공유 계정 같은 기능은 현재 구조가 더 안정화된 뒤 붙이는 편이 좋습니다.

FAQ

자주 묻는 질문

Donggri Ledger는 어떤 구조로 운영되나요?

프론트엔드는 Vercel, 인증과 데이터베이스는 Supabase, 코드와 문서는 GitHub 기반으로 운영합니다.

현재 바로 사용할 수 있는 기능은 어디까지인가요?

회원가입, 로그인, 카드 결제예정, 자산 관리, 고정지출, 예산, 투자 포트폴리오, 백업과 복원을 사용할 수 있습니다.

왜 카드 결제예정 기능이 중요한가요?

카드를 쓴 날과 실제 돈이 빠져나가는 날이 다르기 때문에, 생활비 흐름을 맞추려면 결제예정 금액을 따로 관리해야 합니다.

누가 쓰기 좋은 서비스인가요?

생활비, 카드 결제일, 투자 자산을 한 화면에서 함께 보고 싶은 개인 사용자에게 맞는 구조입니다.
