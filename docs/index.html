로컬 서버를 계속 켜두는 방식 대신, 프론트는 Vercel, 데이터와 인증은 Supabase로 넘겨서 실제 서비스처럼 다루기 쉬운 형태로 전환했습니다.

01

로그인과 계정 복구

아이디와 비밀번호, 보안질문 3개로 가입하고 비밀번호를 재설정할 수 있습니다. 사용자별 데이터는 RLS로 분리됩니다.

02

실제 자금 흐름 중심 가계부

은행, 현금, 카드, 투자계좌를 나눠 관리하고 카드 결제예정 금액을 실제 출금일 기준으로 추적합니다.

03

예산부터 투자까지 연결

고정지출, 예산, 리포트, 검색, 투자 포트폴리오까지 같은 앱 안에서 이어지는 흐름으로 구성되어 있습니다.

Current features

지금 바로 확인할 수 있는 핵심 기능

현재 구현된 기능은 “사용자 인증”, “거래 입력”, “카드 결제예정”, “고정지출/예산”, “투자 포트폴리오”, “운영 편의”로 묶을 수 있습니다.

계정 접근

회원가입, 로그인, 보안질문 기반 비밀번호 찾기, 50명 가입 제한을 포함합니다.

가계부 코어

수입, 지출, 투자 거래 입력과 달력, 검색, 리포트, 월별 요약을 제공합니다.

카드와 자산

카드 자산별 결제일과 결제통장을 두고 카드 결제예정 금액을 계산합니다.

예산과 고정지출

카테고리별 예산을 설정하고 월별 고정지출을 등록해 남은 예산과 미지출 금액을 확인합니다.

투자 포트폴리오

주식, 코인, ETF, 펀드를 분리 관리하고 평가금, 손익, 수익률을 집계합니다.

운영 기능

JSON 백업과 복원, GitHub 기반 버전관리, GitHub Pages 랜딩, 세션 로그 기록을 유지합니다.

Runtime capture

실제 화면에서 어떤 정보가 보이는지

대시보드와 투자 화면 기준으로, 자금 흐름과 포트폴리오가 어떤 식으로 정리되는지 한 번에 확인할 수 있게 구성했습니다.

Recommended roadmap

앞으로 추가하면 좋은 기능

현재 구조를 해치지 않으면서 사용자 가치와 운영 효율을 동시에 올리는 기준으로 다음 기능을 추천합니다.

Priority 1

반복 수입 자동 생성, 계좌 간 이체, 월마감 스냅샷과 순자산 히스토리를 추가하면 매달 반복되는 입력 부담이 크게 줄어듭니다.

Priority 2

예산 초과, 카드 결제일, 고정지출 다가옴을 알려주는 알림 센터를 넣으면 실사용성이 올라갑니다.

Priority 3

CSV 가져오기와 월별 PDF 리포트를 지원하면 실제 마이그레이션과 공유가 쉬워집니다.

Priority 4

투자 가격 이력 저장과 수익률 추세 차트를 만들면 포트폴리오 화면이 단순 조회를 넘어 분석 도구가 됩니다.

Priority 5

가입 현황, Edge Function 상태, 가격 새로고침 실패 로그를 보는 운영 대시보드를 두면 소규모 서비스 운영이 훨씬 편해집니다.

Future option

가족/커플 공유, 태그 기반 거래 분류, 목표저축 같은 기능은 현재 1인 중심 구조가 안정화된 뒤 확장하는 게 좋습니다.

Build story

이 프로젝트는 5단계로 진화했습니다

구현 순서와 왜 그렇게 바꿨는지를 블로그형 기록으로 정리하려는 목적입니다. 현재 허브 페이지는 만들어져 있고, Notion Integration 권한이 열리면 Step 1~5 하위 글을 연결할 예정입니다.

Step 1. 로컬 MVP

FastAPI와 SQLite로 개인 가계부 핵심 흐름을 먼저 만들고, 거래 입력과 자산 관리의 기본 동선을 검증했습니다.

Step 2. 카드 / 투자 확장

카드 결제예정, 고정지출, 예산, 투자 포트폴리오를 붙여 실제 돈 흐름과 자산 흐름을 함께 보도록 넓혔습니다.

Step 3. SEO 랜딩 정리

GitHub Pages 소개 페이지와 검색 유입용 문장을 정리해서 프로젝트 외부 노출의 기반을 만들었습니다.

Step 4. 서비스형 전환

로컬 서버 부담을 줄이기 위해 Supabase + Vercel 조합으로 옮기고, 외부 접속 가능한 구조로 바꿨습니다.

Step 5. 인증과 운영 안정화

아이디/비밀번호, 보안질문, Edge Function, 실시간 가격 새로고침, 배포 오류 대응까지 묶어 서비스 운영 흐름을 안정화했습니다.

FAQ

자주 묻는 질문

지금 이 프로젝트는 어디에 배포되나요?

프론트는 Vercel, 인증과 데이터베이스는 Supabase, 소개 문서와 검색 노출 페이지는 GitHub Pages 기준으로 운영합니다.

이미 동작하는 기능은 어디까지인가요?

회원가입, 로그인, 비밀번호 재설정, 거래 입력, 달력, 고정지출, 예산, 리포트, 투자, 검색, 자산 관리, JSON 백업/복원이 동작합니다.

왜 SMTP 대신 보안질문 기반 복구를 썼나요?

도메인과 메일 운영 복잡도를 줄이면서도 비밀번호 재설정 흐름을 유지하기 위해 현재는 보안질문 기반 복구를 선택했습니다.

다음에 가장 먼저 붙이면 좋은 기능은 무엇인가요?

반복 수입, 계좌 간 이체, 월마감 스냅샷, 알림 센터 순서로 추천합니다. 실제 사용 빈도와 운영 효율이 가장 크게 올라가기 때문입니다.
