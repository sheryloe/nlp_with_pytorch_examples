# Donggri Ledger 명세서

## 1. 프로젝트 개요

- 프로젝트명: Donggri Ledger
- 목적: 개인 가계부를 서비스형 웹앱으로 운영하면서 자산, 카드 결제예정, 예산, 투자까지 한 곳에서 관리
- 운영 구조: `GitHub`로 코드 관리, `Vercel`로 웹 배포, `Supabase`로 인증/DB/함수 처리, `GitHub Pages`로 소개 페이지 운영
- 사용자 모델: 소규모 비공개 서비스, 최대 50명 가입 제한

## 2. 현재 운영 구조

- 프론트엔드: 정적 웹 앱
- 배포: `Vercel`
- 인증: `Supabase Auth`
- 데이터베이스: `Supabase Postgres`
- 데이터 보호: `RLS(Row Level Security)`
- 서버 자동화: `Supabase Edge Functions`
- 문서/랜딩: `README.md`, `GitHub Pages`, `docs/SESSION_LOG.md`

## 3. 지금까지 구현한 기능

### 3-1. 인증과 계정

- 아이디 + 비밀번호 기반 로그인
- 회원가입 시 보안질문 3개 선택 및 답변 저장
- 보안질문 답변 확인 후 비밀번호 재설정
- 회원가입 인원 50명 제한
- 사용자별 데이터 분리
- `RLS` 적용으로 본인 데이터만 조회/수정 가능

## 4. 지금까지 수행한 주요 업무

- 로컬 FastAPI + SQLite 기반 구조 확인
- Supabase + Vercel 중심 서비스형 구조로 전환
- SQLite 의존 로직을 Supabase 스키마/함수 기준으로 재설계
- 사용자 분리용 `user_id` 체계 적용
- Edge Functions 기반 회원가입 / 비밀번호 찾기 / 투자 시세 갱신 흐름 구축
- 투자 모델을 `market`, `currency`, `fx_rate_krw`, `price_source` 기반 멀티통화 구조로 확장
- Yahoo 검색 Edge Function으로 심볼 검색/선택형 투자 입력 UX 구현
- GitHub Pages 소개 페이지를 서비스 소개 중심으로 반복 개선
- 레이아웃, 타이포, 강조 위계, 포털형 구조 조정
- Step 6 SEO형 문서를 Notion, README, GitHub Wiki에 동기화
- Codex 작업 기록을 장기 보관용 아카이브 문서로 정리

## 5. 현재 문제점

- `refresh-market-prices` Edge Function은 실제 운영 환경에서 재배포/실사용 검증을 한 번 더 확인해야 함
- 외부 시세 API 의존 구간은 응답 실패나 지연 가능성이 있음
- `search-market-symbols` 결과 품질과 시장/통화 표기 정합성은 실사용 검증이 더 필요함
- 해외 종목 평균매수가 입력 단위가 원화인지 원통화인지 사용자가 헷갈릴 수 있음
- 자동 테스트가 부족해서 UI 회귀를 수동 확인에 많이 의존하고 있음
- 운영용 에러 모니터링과 가입 현황 대시보드가 없음
- GitHub Pages 캡처 이미지는 앞으로 더 고품질 전체화면 샷으로 교체할 여지가 있음
